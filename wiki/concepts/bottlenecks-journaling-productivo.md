# Bottlenecks como Journaling Productivo

## Principio

Un **bottleneck** es cualquier cosa que te impide avanzar en un proyecto — falta de información, dependencia bloqueada, decisión pendiente, miedo, herramienta faltante. El journaling tradicional los captura para introspección; el journaling productivo los captura para **generar tasks de desbloqueo**.

La diferencia clave: el output del journal no es solo texto — son **acciones concretas** que el sistema te empuja a crear.

## El flujo

1. **Por cada proyecto activo**, ten una sección "Bottlenecks" visible
2. **Cuando te trabas** (en el momento, no en retrospectiva), abres esa sección y escribes qué te bloquea
3. **El acto de escribir** suele revelar la solución — "necesito una base de datos académica" emerge cuando escribes "no encuentro estudios relevantes"
4. **Cada bottleneck se convierte en task**: explícitamente, en el mismo flow, agregas la task al project
5. **La task se prioriza** y se hace antes de continuar el trabajo bloqueado

## Por qué funciona

- **Externaliza el loop mental**: el bloqueo se vuelve visible, no te comes la cabeza en silencio
- **Previene journaling contemplativo**: el journal se conecta al sistema de tasks, no es un sumidero
- **Sella el ciclo**: bottleneck → task → resolución → bottleneck nuevo (o cierre)
- **Filtra por project**: solo ves bottlenecks del proyecto actual, no de toda tu vida

## Ejemplo concreto (del video)

> "Estoy haciendo un report. No encuentro estudios relevantes."
> → Task creada: "Suscribirse a una base de datos académica (Google Scholar / JSTOR)"

Sin el bottleneck journaling, esta task se queda dando vueltas en la cabeza por días. Con él, la escribes una vez, la task existe, la haces, vuelves al report.

## Diferencia con journaling común

| Journaling contemplativo | Bottleneck journaling |
|-------------------------|------------------------|
| "¿Qué siento hoy?" | "¿Qué me bloquea en el proyecto X?" |
| Output: párrafo reflexivo | Output: 1-N tasks concretas |
| Periodicidad: diario | Periodicidad: cuando hay bloqueo |
| Vive en: diario personal | Vive en: página del proyecto |
| Métrica: consistencia de escritura | Métrica: bottlenecks resueltos |

## Cuándo usarlo
- Proyectos largos donde aparecen bloqueos no obvios
- Trabajos de investigación / writing donde la información es el cuello
- Sprints que se estancan a mitad de camino
- Cuando el journaling diario se vuelve rutinario y no genera acción

## Cuándo NO abusar
- No todo pensamiento necesita ser una task — filter signals vs. noise
- Si el journal se vuelve procrastinación disfrazada, está fallando
- Bottlenecks emocionales profundos probablemente requieren conversación humana, no más tasks
